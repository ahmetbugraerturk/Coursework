## Q1: All Possible Strings

In this question, your task to is to find all the possible strings that can be generated by removing some of the letters of an input string, without changing the order of the input string.

Additionally, you need to sort the output list in descending order of string lengths. Strings of the same length can appear in any order.

For example, given the string `'che'`, we can remove the letter 'h' and obtain `'ce'`, or remove the letters 'c' and 'h' and obtain `'e'` and so on. Eventually, we can generate the following strings: [`'che'`, `'ch'`, `'ce'`, `'c'`, `'he'`, `'h'`, `'e'`, `''`]. 

Note that given an string of length `n`, we can remove 0, 1, 2, ..., n letters. (removing 0 letters gives us the original string and removing all the n letters gives us `''`)

Your task is to write the function `generate_all_strings(letter_sequence)`, that takes a string as input, and returns all the possible strings that can be generated as described above, **as a list**.


* Assume that there are **no duplicate letters** in the input string.
* The order of the strings in the returned list **DOES** matter. For example, for the input `'a'` you should return `['a', '']` not `['', 'a']`.


#### Your solution:
*  **MUST** use **RECURSION**. 
*  **MUST NOT** use global variables.
*  **MUST NOT** use any libraries.
* Otherwise, your solution **WILL NOT** be accepted.


**Examples**
```

generate_all_strings('') -> ['']
generate_all_strings('g') -> ['g', '']
generate_all_strings('cd') -> ['cd', 'c', 'd', '']
generate_all_strings('abc') -> ['abc', 'ab', 'ac', 'bc','a', 'b', 'c', '']
```

## Q2: Literal Crush 

In this question, you are going to simulate Candy Crush for letters in 1D.
For the question, you'll be given a sequence of letters.  Once simulation starts, any identical consecutive letters starting from left will disappear. For example,  
  * For the sequence "aabdc" the letters *a*s in the beginning are the same. Hence they will disapper and sequence
    becomes "bdc". Then since there is no any other consecutive same letters, the simulation will stop.
  * Note that any number of consecutive same letters will disappear in one step. That is,  
    The sequence "baaaac" will become "bc"
  * Note that **crushing starts from left**. That is, "abbccba" first becomes "accba". Then becomes "aba".

To do so, write a function named `simulate(letter_sequence)` which takes a letter sequence as input. What you should do is:
* If there are consecutive identical letters, they will crush starting from left and will disappear.
* This will continue untill there is no consecutive identical letters.
* At the end, if all letters are disappeared you should return *Literal Crush!*. Otherwise, you should return remaining letters without changing the order.
* The simulation is case-sensitive, meaning 'A' and 'a' are treated as different letters. For the sequence "AAbbCc", the order will be as follows: "AAbbCc" -> "bbCc" -> "Cc". 




#### Your solution:
*  **MUST** use **RECURSION**. 
*  **MUST NOT** use global variables .
*  You **MUST NOT** use any libraries.
*  Otherwise, your solution **WILL NOT** be accepted.

**Examples**
```
simulate("abbc") -> ac
simulate("abcddcba") -> Literal Crush!
simulate("abcde") -> abcde
simulate("abcddcbba") -> Literal Crush!
  Explanation: "abcddcbba" -> "abccbba" -> "abbba" -> "aa" -> Literal Crush!
simulate("AaBBcd) -> "Aacd"
```
